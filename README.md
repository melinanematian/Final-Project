# پروژه فروشگاه اینترنتی لوازم‌التحریر — pink shop

این پروژه یک فروشگاه اینترنتی ساده برای لوازم‌التحریر است که با زبان Python و کتابخانه `Tkinter` پیاده‌سازی شده است. برنامه دارای دو نقش اصلی است: مشتری و مدیر فروشگاه. مشتری می‌تواند وارد حساب خود شود، حساب جدید بسازد، محصولات را مشاهده کند، جستجو و فیلتر انجام دهد، جزئیات محصول را ببیند، کالا را به سبد خرید اضافه کند، موجودی حساب خود را افزایش دهد، خرید را نهایی کند، سفارش‌های خود را مشاهده کند و برای محصولات کامنت ثبت کند. مدیر فروشگاه نیز می‌تواند محصولات را مدیریت کند، محصول جدید اضافه کند، تصویر محصول انتخاب کند، قیمت، تخفیف، موجودی و توضیحات را ویرایش کند، محصول را حذف کند، سفارش‌ها را ببیند و گزارش فروش دریافت کند.

نام فروشگاه در برنامه:

```text
pink shop
```

---

## اطلاعات ورود ادمین برای بررسی پروژه

برای بررسی بخش مدیریت فروشگاه، می‌توان از حساب زیر استفاده کرد:

```text
username: admin
password: admin123
```

بعد از ورود، برنامه نقش کاربر را از فایل `users.csv` می‌خواند. چون نقش این حساب `admin` است، پنل مدیریت فروشگاه باز می‌شود.

اطلاعات ورود ادمین در صفحه ورود برنامه نمایش داده نمی‌شود تا کاربران عادی به اطلاعات مدیریت دسترسی مستقیم نداشته باشند.

---

## روش اجرای پروژه

فایل اصلی برنامه `main.py` است. برای اجرای پروژه، ابتدا فایل zip را از حالت فشرده خارج کنید و سپس پوشه اصلی پروژه را در محیط برنامه‌نویسی موردنظر باز کنید. فایل‌های داخل پوشه‌های `panels`، `data` و `images` وابسته به برنامه هستند و نباید به صورت جداگانه اجرا شوند.

برای اجرا، فایل زیر را باز و Run کنید:

```text
main.py
```

پوشه اصلی پروژه باید شامل فایل‌ها و پوشه‌های زیر باشد:

```text
main.py
models.py
data_manager.py
panels/
data/
images/
README.md
GUIDE.md
```

## فایل‌های اصلی برنامه

### `main.py`

این فایل نقطه شروع برنامه است و پنجره اصلی را با `Tkinter` می‌سازد. صفحه ورود، ثبت‌نام مشتری، تشخیص نقش کاربر و انتقال به پنل مناسب از طریق این فایل مدیریت می‌شود.

وظایف اصلی این فایل:

- ساخت پنجره اصلی برنامه
- تنظیم عنوان برنامه روی `pink shop`
- طراحی صفحه ورود با تم صورتی
- طراحی صفحه ثبت‌نام مشتری
- اتصال برنامه به `CSVManager`
- بررسی نام کاربری و رمز عبور
- تشخیص نقش کاربر از فایل `users.csv`
- باز کردن پنل مشتری برای نقش `customer`
- باز کردن پنل ادمین برای نقش `admin`
- جلوگیری از ساخت حساب ادمین توسط کاربران عادی

---

### `models.py`

این فایل شامل کلاس‌های اصلی پروژه است و ساختار شیءگرای برنامه را تشکیل می‌دهد.

کلاس‌های اصلی:

```text
User
Product
CartItem
Order
Comment
```

کلاس `User` اطلاعات کاربر را نگهداری می‌کند. کلاس `Product` برای اطلاعات محصولات استفاده می‌شود. کلاس `CartItem` آیتم‌های سبد خرید را نمایش می‌دهد. کلاس `Order` سفارش‌های ثبت‌شده را مدل‌سازی می‌کند و کلاس `Comment` برای کامنت‌های کاربران روی محصولات استفاده می‌شود.

---

### `data_manager.py`

این فایل مسئول مدیریت فایل‌های CSV است. تمام عملیات خواندن، نوشتن، پیدا کردن، ذخیره و به‌روزرسانی داده‌ها از طریق کلاس `CSVManager` انجام می‌شود. این کار باعث می‌شود کدهای پروژه تمیزتر، قابل‌فهم‌تر و قابل‌توسعه‌تر باشند.

وظایف اصلی:

- خواندن و ذخیره کاربران
- بررسی ورود کاربر
- خواندن و ذخیره محصولات
- ذخیره سفارش‌ها
- خواندن سفارش‌ها
- ذخیره و خواندن کامنت‌ها
- محاسبه گزارش فروش

## فایل‌های داده

### `users.csv`

این فایل اطلاعات کاربران را ذخیره می‌کند.

```text
username,password,balance,role
```

نقش کاربر می‌تواند `customer` یا `admin` باشد.

### `products.csv`

این فایل اطلاعات محصولات را ذخیره می‌کند.

```text
product_id,name,price,stock,category,discount,description,image,colors
```

### `orders.csv`

این فایل سفارش‌های ثبت‌شده را ذخیره می‌کند.

```text
username,product_id,quantity,total_price,date,selected_color
```

### `comments.csv`

این فایل کامنت‌های کاربران روی محصولات را ذخیره می‌کند.

```text
username,product_id,comment,date
```
