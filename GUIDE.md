# راهنمای پروژه فروشگاه اینترنتی لوازم‌التحریر — pink shop

این فایل راهنمای کامل پروژه `pink shop` است و ساختار برنامه، نقش فایل‌ها، منطق اجرای بخش‌ها، تقسیم وظایف گروهی و امکانات اصلی و امتیازی پروژه را توضیح می‌دهد. پروژه با Python و Tkinter پیاده‌سازی شده و داده‌های آن در فایل‌های CSV ذخیره می‌شوند.

---

## 1. معرفی کلی پروژه

`pink shop` یک فروشگاه اینترنتی ساده برای لوازم‌التحریر است. برنامه دو نوع کاربر دارد:

```text
customer
admin
```

مشتری می‌تواند محصولات را مشاهده کند، محصول موردنظر را جستجو یا فیلتر کند، جزئیات و عکس محصول را ببیند، محصول را به سبد خرید اضافه کند، موجودی حساب خود را افزایش دهد، خرید را نهایی کند، سفارش‌های خود را ببیند و برای محصولات کامنت ثبت کند.

ادمین می‌تواند محصولات را مدیریت کند، محصول جدید اضافه کند، عکس محصول انتخاب کند، قیمت، موجودی، تخفیف و توضیحات محصول را تغییر دهد، محصول را حذف کند، سفارش‌ها را مشاهده کند و گزارش فروش بگیرد.

## 5. فایل `main.py`

فایل `main.py` نقطه شروع برنامه است. وقتی پروژه اجرا می‌شود، ابتدا این فایل اجرا شده و پنجره اصلی برنامه را می‌سازد.

کلاس اصلی برنامه:

```python
class OnlineStoreApp(tk.Tk):
```

این کلاس از `tk.Tk` ارث‌بری می‌کند؛ بنابراین خودش پنجره اصلی برنامه Tkinter است.

وظایف اصلی این کلاس:

- ساخت پنجره اصلی
- تنظیم اندازه پنجره
- تنظیم عنوان برنامه
- ساخت container برای جابه‌جایی بین صفحه‌ها
- ساخت شیء `CSVManager`
- نگهداری کاربر فعلی در `self.current_user`
- نمایش صفحه ورود
- نمایش پنل مشتری
- نمایش پنل ادمین
- مدیریت خروج از حساب کاربری

## 6. اتصال برنامه به فایل‌های CSV

در `main.py`، برنامه با این خط به سیستم مدیریت داده‌ها وصل می‌شود:

```python
self.data_manager = CSVManager(data_dir="data")
```

کلاس `CSVManager` داخل فایل `data_manager.py` قرار دارد. این کلاس مسئول خواندن و نوشتن داده‌ها در فایل‌های CSV است.

فایل‌های داده:

```text
users.csv
products.csv
orders.csv
comments.csv
```

این طراحی باعث می‌شود بخش‌های مختلف برنامه مستقیماً درگیر جزئیات باز کردن، خواندن و ذخیره فایل‌ها نشوند. در نتیجه کد تمیزتر و قابل‌توسعه‌تر است.

## 7. صفحه ورود و ثبت‌نام

صفحه ورود در کلاس زیر ساخته شده است:

```python
class LoginPage(ttk.Frame):
```

این صفحه شامل موارد زیر است:

- عنوان فروشگاه
- کادر username
- کادر password
- دکمه ورود
- دکمه ساخت حساب مشتری

فرآیند ورود:

1. کاربر username و password را وارد می‌کند.
2. برنامه اطلاعات را با فایل `users.csv` بررسی می‌کند.
3. اگر اطلاعات درست باشد، نقش کاربر خوانده می‌شود.
4. اگر نقش `customer` باشد، پنل مشتری باز می‌شود.
5. اگر نقش `admin` باشد، پنل ادمین باز می‌شود.
6. اگر اطلاعات اشتباه باشد، پیام خطا نمایش داده می‌شود.

صفحه ثبت‌نام فقط حساب `customer` می‌سازد. بنابراین کاربر عادی نمی‌تواند برای خودش حساب ادمین ایجاد کند.

insertbackground="#FF1493"

## 12. سبد خرید

در پنل مشتری، سبد خرید با یک لیست مدیریت می‌شود:

```python
self.cart = []
```

هر آیتم سبد خرید از کلاس `CartItem` ساخته می‌شود. هر آیتم اطلاعاتی مثل محصول، تعداد و رنگ انتخاب‌شده را نگه می‌دارد.

منطق کلی خرید:

1. مشتری محصول را انتخاب می‌کند.
2. تعداد محصول وارد می‌شود.
3. برنامه موجودی محصول را بررسی می‌کند.
4. محصول به سبد خرید اضافه می‌شود.
5. هنگام نهایی کردن خرید، موجودی حساب مشتری بررسی می‌شود.
6. در صورت کافی بودن موجودی، سفارش ثبت می‌شود.
7. موجودی محصول کم می‌شود.
8. موجودی حساب مشتری کم می‌شود.
9. سفارش در `orders.csv` ذخیره می‌شود.
